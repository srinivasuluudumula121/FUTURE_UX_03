# Information Architecture

## Product Structure

ClientFlow CRM was structured to help small agencies manage leads, clients, follow-ups, and daily operations in one centralized system.

The architecture prioritizes:

* operational visibility
* workflow clarity
* fast information scanning
* task prioritization
* scalability

## Main Navigation

### Dashboard

Agency overview, KPIs, urgent actions, and quick visibility.

### Lead Pipeline

Visual kanban system to track leads across stages.

### Client Management

Active client list and project visibility.

### Client Profile

Detailed client information, engagement tracking, notes, and activity.

### Tasks & Follow-Ups

Task organization and reminder management.

### Analytics

Performance overview and operational insights.

### Settings

CRM configuration and preferences.

## Navigation Structure

Dashboard
→ Lead Pipeline
→ Client Management
→ Client Profile
→ Tasks & Follow-Ups
→ Analytics

## Design Goal

The architecture was intentionally designed to support daily agency operations while reducing workflow confusion.
